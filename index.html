🔍 Voynich-Manuskript – Entschlüsselung

Ein altes Rätsel. Klare Methode. Offener Code.

📜 Überblick

Ich dokumentiere hier meinen praktischen Ansatz zur Entschlüsselung des Voynich-Manuskripts. Kein Forschungskollektiv, kein PR-Team – nur eine saubere Methode, reproduzierbare Ergebnisse und ein Repository, das jeder prüfen kann.

Status: laufende Arbeit

  • Transkription & Symbolinventar
  • Häufigkeits- & Musteranalyse
  • Substitution + Constraints
  • Kontextvalidierung

🧠 Methode

Der Workflow ist modular aufgebaut: von der Datenerhebung über statistische Modelle bis hin zur iterativen Übersetzung mit Gegenprüfung im historischen Kontext. Die komplette Beschreibung liegt in docs/methodology.md.

# Minimalbeispiel
python src/decrypt.py examples/sample_input.txt

📊 Ergebnisse (Work-in-Progress)

Aktuell sind etwa ~65 % des Materials stabil lesbar. Thematische Cluster: Kräuterheilkunde, Astronomie, Rezepturen. Beispiele und Notizen unter docs/results.md.

💻 Code & Struktur

git clone https://github.com/Tim-K1984/voynich-decryption.git
cd voynich-decryption
python src/decrypt.py examples/sample_input.txt
src/        → Analyse & Entschlüsselung
data/       → Quellmaterial & Mapping-Tabellen
docs/       → Forschung & Dokumentation
examples/   → Beispiel I/O

Mapping-Dateien sind schlank und nachvollziehbar gehalten. Änderungen erfolgen versioniert über PRs/Issues.

🔍 Reproduzierbarkeit

  • Alle Skripte deterministisch, seeds fixiert (wo relevant).
  • Eingangsdatensätze und Mappings im Repo.
  • Jede Übersetzung wird mit Kontextquellen kommentiert (siehe docs/).

Fragen, Anregungen, Gegenbeweise? Immer her damit. Pull Requests willkommen – die Verantwortung für Aussagen und Interpretation liegt aber bei mir.
